**Let's do it, step by step:**

1. 
```bash
$ curl -s https://ctf:88sPVWyC2P3p@challenge01.cloud-champions.com/actuator/mappings | jq '.contexts.spring.mappings.dispatcherServlets.dispatcherServlet[]'
```
#### > Focus: Whenever, we are targeting a list for listing, we are using `[]`.

<img width="2677" height="1086" alt="image" src="https://github.com/user-attachments/assets/8f33a04d-e040-481d-b88d-451ee1095cf5" />

2. 
```bash
$ curl -s https://ctf:88sPVWyC2P3p@challenge01.cloud-champions.com/actuator/mappings | jq '.contexts.spring.mappings.dispatcherServlets.dispatcherServlet[]?.details'
```
#### > Focus: Here, to access `details`. but why `?` is used ? => 
> It tells to `jq`, if this element or array exists, access it â€” otherwise, just skip it silently.

<img width="2081" height="968" alt="image" src="https://github.com/user-attachments/assets/3756cb87-6646-49d5-9cdf-c1cfe85c5df4" />

```bash
$ curl -s https://ctf:88sPVWyC2P3p@challenge01.cloud-champions.com/actuator/mappings | jq '.contexts.spring.mappings.dispatcherServlets.dispatcherServlet[]?.details?.requestMappingConditions?.patterns[]'
```
#### > Focus: Here, to access `requestMappingConditions` and `patterns` (`patterns` is a list, so to list it, use `[]`). the logic behind use of '?' is same.
